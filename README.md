# Colon Cancer Severity Assessment System

A comprehensive clinical decision support system for colon cancer screening, severity assessment, and genomic driver/passenger gene classification. The system combines classical ML models, deep learning, and a clinical severity engine to provide actionable insights from patient data.

## 🧩 System Overview

### Classical ML Pipeline (`FYP.py`)

Trains and evaluates 5 regularized models on `ML_READY_DATA.csv`:

| Model | Key Parameters |
|---|---|
| Logistic Regression | L2 penalty, C=0.01, class_weight='balanced' |
| Decision Tree | max_depth=3, min_samples_leaf=15 |
| Random Forest | n_estimators=150, max_depth=4, min_samples_split=25 |
| SVM (Linear) | C=0.005, class_weight='balanced' |
| XGBoost | reg_lambda=50, reg_alpha=5, learning_rate=0.03 |

Outputs: 4 visualization PNGs + `trained_rf_model.pkl` + `dataset_scaler.pkl`

---

### Deep Learning Pipeline (`FYP_DL.py`)

Trains a 2-hidden-layer MLP (64 → 32 → 1) on feature-selected and scaled data:

- **Feature Selection**: ANOVA SelectKBest (k=25)
- **Regularization**: L2 alpha=0.08, early_stopping with 15% validation
- **Imbalance Correction**: Balanced sample weights
- **Outputs**: Loss curve + confusion matrix PNGs + `trained_mlp_model.pkl`

---

### Genomic Analysis Pipeline (`genomic1.py`)

Ranks all genes in `advanced_genomic_data.csv` by AI-derived danger score using XGBoost:

- Groups by gene name, takes mean danger score
- Outputs top-10 and top-100 rankings to JSON for the Genomic UI

---

### Clinical Severity Engine (`severity_engine.py`)

Determines patient severity in 3 steps:

1. **Base Level** from cancer probability thresholds:
   - Mild: 0.00–0.30 | Moderate: 0.30–0.55 | High: 0.55–0.75 | Critical: 0.75–1.00

2. **Clinical Adjustment** (±1 level) based on 12 clinical signals:
   - Upgrades: metastasis, G3/G4 grade, R1/R2 residual disease, recurrence, prior malignancy, short follow-up, age >70
   - Downgrades: R0 clean margins, no metastasis, G1 grade, no family history, non-smoker, long follow-up

3. **Treatment Recommendation** per severity level

---

### GUI Applications

#### `New_Patient_UI.py`
Two-panel CustomTkinter GUI:
- **Left panel**: Patient data entry form (age, gender, tumor grade, metastasis, residual disease, tumor classification, prior malignancy, family history, smoking, follow-up days)
- **Right panel**: Color-coded results — cancer probability, severity level, clinical adjustment explanation, treatment recommendation
- Saves records to `new_patients.csv`

#### `Genomic_UI.py`
Two-section CustomTkinter GUI:
- **Top section**: Top 10 gene danger scores — horizontal statbars proportional to danger score, color-coded (red ≥0.90, orange ≥0.75, yellow ≥0.55, green <0.55)
- **Bottom section**: Top 100 ranked genes in a scrollable table with color dots

#### `Model_Evaluation_UI.py`
Dashboard with 4 evaluation panels:
- Confusion matrices from all classical models
- Train vs Test accuracy gap line chart
- Precision-Recall scatter plot
- Feature importance bar chart
- Patient triage — enter a patient index to get severity assessment

#### `Deep_Learning_UI.py`
Deep learning dashboard:
- Training loss convergence curve
- MLP confusion matrix
- Live patient triage using the MLP inference pipeline

## 📊 Severity Levels & Treatment Recommendations

| Level | Probability Range | Treatment |
|---|---|---|
| **Mild** | 0.00 – 0.30 | Routine monitoring, screening, lifestyle management |
| **Moderate** | 0.30 – 0.55 | Chemotherapy consultation, clinical observation, periodic imaging |
| **High** | 0.55 – 0.75 | Aggressive chemotherapy, radiation therapy, possible surgery |
| **Critical** | 0.75 – 1.00 | Immediate oncology intervention, targeted therapy, intensive treatment planning |

## ⚠️ Important Notes

- **Run pipelines first**: `genomic1.py`, `FYP.py`, and `FYP_DL.py` must be run before their respective UIs to generate the required `.pkl` and `.json` artifacts.
- **Python version**: Designed for Python 3.12+.
- **sklearn version mismatch**: The pickled models were saved with scikit-learn 1.8.0. Loading with a different version produces warnings but is generally safe.
- **Data files**: All CSV/Excel data files must be present in the same directory as the scripts.
